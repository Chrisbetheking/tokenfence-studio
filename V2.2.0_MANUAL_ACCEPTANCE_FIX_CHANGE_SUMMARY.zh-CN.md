# Chris Studio v2.2.0 人工验收问题修复摘要

本批次根据真实安装包人工测试截图集中修复，不再只处理单个编译报错。

## 修复内容

1. **完整回答被误判为流式失败**
   - Rust SSE 读取在已经收到正文后遇到连接 EOF 时保留正文并正常结束。
   - Renderer 对晚到的 `STREAM_READ_ERROR` 做二次防御，不再把已显示的完整回答转换为红色失败卡片。
   - 新增“先收到正文、随后收到读取错误”的自动化回归测试。

2. **会话无法重命名**
   - 最近对话支持编辑按钮、双击和右键进入内联重命名。
   - Enter 保存，Escape 取消，失焦保存。
   - 历史页使用独立编辑容器，移除输入框嵌套在按钮中的不稳定结构。
   - 重命名写回本地持久化，并同步当前已打开的会话，防止下一次发送消息覆盖新名称。
   - 删除当前会话时自动进入新任务，防止已删除会话被再次保存回来。

3. **Computer Use 权限判断失真**
   - 辅助功能与屏幕录制分开检测。
   - 使用 CoreGraphics 原生 Screen Recording 权限预检和请求。
   - 能力卡片根据当前进程实际权限显示 `ready` 或 `permission-needed`。
   - 请求权限后立即刷新能力状态。

4. **TextEdit 打开成文件选择器、输入焦点错误**
   - 打开 TextEdit 后使用受控 `Command+N` 创建新空白文档。
   - 模型执行输入或按键前重新聚焦最近成功打开的白名单应用。
   - 避免用户确认弹窗关闭后焦点回到 Chris Studio，导致文字输进错误窗口。
   - 补齐 `cmd+w` 前后端白名单一致性。

5. **模型错误宣称 Computer Use 已完成**
   - 非视觉模型禁止请求截图和坐标点击。
   - TextEdit 目标必须至少存在成功打开 TextEdit 和成功输入文字的观察记录。
   - 最近一步失败时拒绝模型直接返回 `done`。
   - 最多 8 步和紧急停止边界保持不变。

6. **依赖锁文件不一致**
   - 修正 UI `package-lock.json` 与 v2.2.0 `package.json` 的依赖解析差异。
   - `npm ci --legacy-peer-deps` 已实际执行通过。
