24## 项目简介
25
26**TokenFence Studio** 是一个面向大语言模型（LLM）的本地优先安全编排工作台，提供 Prompt 安全扫描、文档智能处理、模型矩阵对比和上下文安全路由。
27
28不是又一个聊天界面。核心思路是在用户输入到达 LLM 之前，构建一个可检查、可清洗、可保护、可分块、可路由的 Pre-LLM 层。
29
30## 功能矩阵
31
32| 功能区域 | 能力 | 状态 |
33|---|---|---|
34| Prompt Guard | 检测密钥、凭据、Token、邮箱、手机号、数据库 URL 及危险 Prompt 内容 | 正常 |
35| 脱敏 | 将敏感值替换为安全占位符 | 正常 |
36| 文档智能 | PDF / DOCX / 图片 OCR 解析、清洗和分块 | 正常 / 部分实验 |
37| 输出生成 | Markdown、HTML、JSON、PDF、ZIP-wrapped DOCX | 验收已验证 |
38| 模型矩阵 | 并排对比多个模型响应 | 正常 |
39| 文件级路由 | 按文件类型、风险和任务意图路由 | 正常 |
40| Provider Hub | OpenAI、Claude、Gemini、DeepSeek、通义千问、Kimi、豆包、智谱、Ollama、LM Studio、自定义端点 | 正常；需用户提供密钥 |
41| 本地运行时 | 执行已批准的本地任务并保存日志 | 已验证 |
42| Obsidian 记忆 | 将输出笔记写入 test vault | 已验证 |
43| API 连接器 | 测试真实或模拟 HTTP 连接器 | 已验证 |
44| Computer Use | 权限门控操作流 | 实验功能 |
45| Android Mobile Lite | 移动优先配套应用 | internal APK 已验证 |
46| Windows Desktop | Tauri 桌面应用 | 实验性 i686 构建 |
47| i18n | 英文 / 简体中文 UI 和 README | 正常 |

64## 已知限制
65
66- 这是发布候选版，不是 v1.0 最终生产版本。
67- Windows Desktop 是 unsigned i686 实验性构建。
68- Windows x64 因当前构建环境缺少 MSVC linker / 64-bit MinGW-w64 而被阻塞。
69- macOS artifact 的 CI 已配置但尚未验证。
70- Android 为 Mobile Lite，不是完整桌面对等功能。
71- 不提供 Play Store 生产签名。
72- 提供商调用需要用户提供 API 密钥。
73- Computer Use 完整控制仍为实验功能。
