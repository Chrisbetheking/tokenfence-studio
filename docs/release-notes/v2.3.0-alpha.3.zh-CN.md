# Chris Studio v2.3.0-alpha.3

本次 Alpha 为多模型 Agent 增加显式检查点恢复，不修改已经通过验收的普通对话、Computer Use、会话持久化和可靠执行中心核心链路。

## 新增

- Planner、Executor、Reviewer 或有限修订失败后，会在对应 Agent 回执中显示可恢复阶段。
- 用户必须主动点击恢复按钮；系统不会后台自动重试。
- Executor 检查点恢复会复用已经完成的 Planner 计划，只执行 Executor 与 Reviewer。
- Reviewer 检查点恢复会保留已经完成的 Executor 草稿，只重新运行 Reviewer。
- 有限修订失败后可以从修订阶段继续，不重新生成原始草稿。
- 恢复时锁定原回执中的角色 Provider 配置；配置缺失或停用时明确失败，绝不静默换模型。
- 每个回执最多允许两次检查点恢复，避免无限重试与 Token 消耗。
- 使用过视觉图片的 Executor/修订检查点禁止在缺少图片时恢复，并要求重新添加图片启动新任务。
- 恢复记录、来源运行 ID、阶段和结果随会话持久化。

## 安全边界

- 文件写入、命令、GitHub、MCP 和 Computer Use 审批规则保持不变。
- 检查点恢复需要用户确认，不能由模型自行触发。
- 已完成角色不会重复调用，但任何尚未完成的外部操作仍需要真实工具回执。

## 本次重点验收

1. 制造 Reviewer 临时不可用，确认 Executor 草稿保留并出现“重试审查”。
2. 恢复 Reviewer 后，确认 Planner 与 Executor 不再调用。
3. 角色配置被删除时，确认恢复明确失败且不切换模型。
4. 使用视觉图片的 Executor 任务失败时，确认恢复被阻止并要求重新添加图片。
