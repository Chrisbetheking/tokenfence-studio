# Chris Studio v2.3.0-alpha.4

本版本新增事务式 Coding Agent：先规划并按需读取真实项目文件，生成多文件统一 Diff；用户逐文件选择后才写入。写入前自动创建受保护的 before 快照，写入后创建 after 快照，随后可运行允许的检查、由独立 Reviewer 审查真实 Git Diff 与命令回执，并支持单文件拒绝、整体回滚或最终接受。

回滚会先核对文件是否仍与 Agent 写入后的 after 快照一致；检测到用户或其他工具产生了更新内容时会拒绝覆盖，避免回滚误删新修改。同一项目只能存在一个待处理事务，旧事务未接受或回滚前，前端和 Rust 后端都会拒绝启动下一次写入。事务文件存放在 `.git/chris-studio/transactions/`，不会污染工作区 Git 状态。应用异常退出或重启后会恢复最近一个仍待处理的事务，并允许继续检查、接受或回滚；由于原始模型提示词不写入事务档案，恢复后 Reviewer 会保持禁用，避免脱离原始验收目标进行虚假审查。

安全边界：仅支持已批准 Git 仓库内的常规文本文件 add/modify/delete；阻止 `.git`、内部事务目录、二进制、子模块、符号链接、复制、重命名、文件模式变化、路径穿越、头部路径不一致和超限备份。补丁必须直接从 `diff --git` 开始，且 Chris Studio 会再使用 Git 自身的 `--numstat` 解析结果核对全部实际目标路径，防止在已审查段落后夹带传统 Diff 或隐藏文件修改。新增目录会从最近存在的安全父目录进行范围验证。

发布工作流恢复为先生成架构验证过的 `.app`，再由原生 `hdiutil` 生成 DMG；Intel DMG 失败时保留 APP ZIP。带连字符的 Alpha 标签自动标记为 Pre-release，并且不会覆盖 v2.2.0 的 Latest 稳定版本。
